# Legacy Appendix

This appendix is intentionally non-canonical. It exists so maintainers can keep historical context without mixing retired behavior into the supported runbooks.

## What Is Archived Here

- retired direct-login flows from the pre-Session-Manager era
- old environment installers and `DAYOA`-era bootstrap material
- superseded staging or bootstrap experiments
- historical notes that still help with archaeology, migration review, or file provenance

## Current Rule

If an operator can follow it today without leaving the supported path, it belongs in `README.md` or `docs/`. If it is only useful for code archaeology or migration history, it belongs here.

## Where To Look

- `../2026-04-pre-rewrite/` for the full pre-rewrite doc snapshot
- `../legacy-dayoa-env/` for retired environment materials
- a local `docs_orig/` tree, when present, for old reference docs and presentation patterns
- `../../bin/legacy/` and `../../daylily_ec/resources/payload/quarantine/` for quarantined runtime assets

Do not treat this appendix as an operator runbook.
