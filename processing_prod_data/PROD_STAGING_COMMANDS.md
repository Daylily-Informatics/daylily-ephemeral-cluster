# Production S3 Staging Commands For mk-gotime3

Target cluster: `mk-gotime3` in `us-west-2`.

Source bucket: `s3://lsmc-ssf-sequencing-data`.

Reference/staging bucket: `s3://lsmc-dayoa-omics-analysis-us-west-2`.

## Inventory

Read-only inventory command used to list source objects:

```bash
AWS_PROFILE=lsmc AWS_REGION=us-west-2 aws s3 ls s3://lsmc-ssf-sequencing-data --recursive
```

The normalized inventory is written to:

```text
processing_prod_data/s3_inventory_fastq_cram.tsv
processing_prod_data/s3_inventory_kind_counts.tsv
processing_prod_data/staging_summary.tsv
```

The saved inventory keeps source-relevant rows only: `CRAM`, `CRAI`, `FASTQ`, and metadata references. Ignored non-source objects are counted in `s3_inventory_kind_counts.tsv` but are not saved in the detailed TSV.

## Manifest Rules

CRAM rows are included only when the exact sibling CRAI exists in source S3:

```text
s3://.../sample.cram
s3://.../sample.cram.crai
```

Rows are excluded for `_unmatched`, `TT`, `KPF*`, `PhiX*`, non-primary CRAM names, and CRAMs without exact sibling `.cram.crai`.

Illumina manifests are not created unless paired R1/R2 FASTQ objects are found. This inventory found `illumina_fastq_runs=0`.

## Validation

Verify every generated manifest row has an exact sibling CRAI in inventory:

```bash
python - <<'PY'
from pathlib import Path
import csv

inventory = Path("processing_prod_data/s3_inventory_fastq_cram.tsv")
keys = set()
with inventory.open() as handle:
    for row in csv.DictReader(handle, delimiter="\t"):
        keys.add(row["key"])

failures = []
for path in sorted(Path("processing_prod_data").glob("ultima_*_analysis_sample_manifest.tsv")):
    with path.open() as handle:
        for row in csv.DictReader(handle, delimiter="\t"):
            key = row["ULTIMA_CRAM"].split("s3://lsmc-ssf-sequencing-data/", 1)[1]
            if f"{key}.crai" not in keys:
                failures.append((str(path), key))

if failures:
    raise SystemExit(f"missing exact sibling .cram.crai entries: {failures[:10]}")
print("all generated CRAM rows have exact sibling .cram.crai entries")
PY
```

Validate manifests through the staging parser:

```bash
source ./activate
python - <<'PY'
from pathlib import Path
from daylily_ec.stage_samples import build_aws_env, load_manifest_rows, AwsConfig

aws_env = build_aws_env(AwsConfig(profile="lsmc", region="us-west-2"))

    print(path.name, len(rows))
PY
```

## Staging

Stage each manifest with the plain staging command. No CRAI files are generated by the CLI; CRAIs are copied from source S3 after each CRAM.

```bash
source ./activate
for manifest in processing_prod_data/ultima_*_analysis_sample_manifest.tsv; do
  stem="$(basename "$manifest" .tsv)"
  outdir="processing_prod_data/staged/$stem"
  mkdir -p "$outdir"
  daylily-ec samples stage "$manifest" \
    --profile lsmc \
    --region us-west-2 \
    --reference-bucket s3://lsmc-dayoa-omics-analysis-us-west-2 \
    --config-dir "$outdir" \
    --stage-target /data/staged_sample_data \
    >"$outdir/stage_stdout.txt" 2>"$outdir/stage_stderr.txt"
done
```

After staging, parse each `stage_stdout.txt` for:

```text
Remote FSx stage directory: /fsx/data/staged_sample_data/remote_stage_<timestamp>
Generated configuration files:
  samples.tsv -> s3://...
  units.tsv   -> s3://...
```

## Workflow Handoff

Use `hg38_broad`, equivalent to `dy-a slurm hg38_broad`.

Ultima alignstats command:

```bash
daylily-ec workflow launch \
  --cluster mk-gotime3 \
  --region us-west-2 \
  --profile lsmc \
  --genome hg38_broad \
  --destination <ULTIMA_RUN_ID> \
  --git-tag main \
  --stage-dir <REMOTE_FSX_STAGE_DIR> \
  --dy-command "bin/day_run produce_alignstats --config dppl=['na'] -p -j 20 -k"
```

Use one `day-clone -d <ULTIMA_RUN_ID>` destination per Ultima run. The remote
analysis checkout should resolve to:

```text
/fsx/analysis_results/ubuntu/<ULTIMA_RUN_ID>/daylily-omics-analysis
```

Do not launch multiple production Ultima runs into a shared `dayoa`
destination, because each launch writes `config/samples.tsv` and
`config/units.tsv` inside the selected analysis checkout.
